# خطة نظام المراسلات الداخلية

## 1) قراءة المشروع وتحديد نقاط الدمج
- الواجهة الخلفية مبنية على FastAPI مع SQLAlchemy و Alembic.
- نمط المشروع الحالي يعتمد تقسيمًا واضحًا: `models/` و `schemas/` و `api/`.
- نظام الصلاحيات الحالي يوفر `get_current_internal_user` المناسب للمراسلات الداخلية بين الموظفين.

## 2) خطة التنفيذ المرحلية
1. **المرحلة الأولى (الأساسات)**
   - إنشاء جداول قاعدة البيانات: المحادثات، المشاركون، الرسائل.
   - دعم نوع المحادثة: مباشر/جماعي.
   - إضافة API أولية: إنشاء محادثة، عرض المحادثات، عرض تفاصيل محادثة، إرسال رسالة.
2. **المرحلة الثانية (التحسينات التشغيلية)**
   - إضافة ترقيم صفحات للرسائل.
   - تحسين عداد غير المقروء بأداء أعلى عبر استعلامات مجمعة.
   - إضافة أرشفة/إغلاق محادثة.
3. **المرحلة الثالثة (التكامل الكامل)**
   - ربط النظام مع الإشعارات.
   - واجهة Frontend للمحادثات المباشرة والجماعية.
   - دعم التحديث الفوري (WebSocket أو SSE).

## 3) ما تم وضعه الآن (الأساسات)
- تم إنشاء نموذج بيانات كامل للمراسلات الداخلية.
- تم إنشاء migration لقاعدة البيانات.
- تم إضافة مسارات API الأساسية وربطها بالتطبيق الرئيسي.

## 4) بداية التنفيذ الفعلي
بدأ التنفيذ الفعلي من خلال بناء نواة قابلة للاستخدام والتوسعة مباشرة في backend.
